# 开发路线图（12 周 / M1–M6）

> 本文档给出 12 周六阶段（M1–M6）的目标、交付物与依赖关系，MVP 压缩路径，以及风险与对策表。
> 团队规模 8 人，按模块并行推进；所有接口以 architecture.md / agent-design.md 的约定为准，先骨架后实现。

---

## 1. 阶段总览

| 阶段 | 周次 | 名称 | 一句话目标 |
| --- | --- | --- | --- |
| M1 | W1–W2 | 离线大数据管道 | 语料采集→清洗→句对齐→术语抽取跑通，产出初始术语库/TM/向量库 |
| M2 | W3–W4 | 单 Agent 基线 | LLMClient + 单 LLM 直译基线可跑，作为实验对照组 B2 |
| M3 | W5–W7 | 多智能体骨架 | 六 Agent 骨架 + MemoryHub + LangGraph QA 回环全链路打通 |
| M4 | W8–W9 | 系统联调与增强 | RAG 注入调优、回环策略调参、成本与稳定性优化 |
| M5 | W10–W11 | 评测实验 | 基线对比 + 消融矩阵 + COMET/LLM-Judge/MQM 全套实验 |
| M6 | W12 | 论文与答辩 | 撰写报告/论文、整理实验图表、答辩材料 |

## 2. 各阶段详表

### M1 离线大数据管道（W1–W2）

- **目标**：按 data-pipeline.md 跑通四步，产出可入库的 JSONL 句对与术语表。
- **交付物**：
  - `mant.pipeline` 四步骨架（collect / clean / align / extract_terms）+ CLI 入口；
  - 至少 3 部作品的双语语料（合规来源）、句对 ≥ 2 万条、术语 ≥ 500 条；
  - 清洗报告与对齐抽样质检记录。
- **依赖**：无（启动项）；vecalign/LASER 可选安装，缺失走长度对齐降级。
- **验收**：JSONL 通过 schema 校验脚本；抽样对齐准确率 ≥ 95%。

### M2 单 Agent 基线（W3–W4）

- **目标**：`LLMClient`（fast/strong 双档、`[DRAFT]` 占位）+ 单 Prompt 直译脚本，产出对照组译文。
- **交付物**：
  - `mant.llm.client` 骨架与配置加载（`llm.providers.*`）；
  - 基线翻译 CLI：输入章节 → 输出译文 + token 用量日志；
  - 3 部测试作品各 1 章的基线译文样例。
- **依赖**：M1 提供测试用原文；API key（缺失时用 `[DRAFT]` 占位跑通流程）。
- **验收**：无 key 环境 `python -m unittest` 通过；有 key 环境 1 章直译成功并记录成本。

### M3 多智能体骨架（W5–W7，关键路径）

- **目标**：按 architecture.md 打通"术语→翻译→审校→润色→QA"主流程与 QA 回环。
- **交付物**：
  - `mant.agents` 六 Agent 骨架（含 Prompt 模板与 output 键契约）；
  - `mant.memory`：models（dataclass）+ MemoryHub（SQLite 实现）+ FAISS 适配（可降级）；
  - `mant.workflow`：`TranslationState` + LangGraph 图（条件边：`qa_verdict` × `rework_count`）；
  - 端到端 demo：1 章翻译全流程跑通，回环至少触发一次（可用低 QA 阈值人为制造）。
- **依赖**：M1 的术语库/TM 数据；M2 的 LLMClient。W5 必须先冻结接口（团队约定文档为唯一事实来源）。
- **验收**：无 key 环境全链路 `[DRAFT]` 模式跑通；有 key 环境 1 章质量可人工检视；`max_rework` 兜底生效。

### M4 系统联调与增强（W8–W9）

- **目标**：把"能跑"变成"跑得好、跑得稳、跑得起"。
- **交付物**：
  - RAG 注入策略调优（`search_tm` 的 k、相似度阈值、圣经摘要长度）；
  - QA 阈值与回环落点调参（当前通过线 7.0/10、回退到翻译 vs 审校的对比小实验）；
  - 稳定性：重试、超时、Agent 失败降级路径；并发批跑 5 章以上不崩；
  - 成本看板：token 用量与 ¥/章 统计脚本。
- **依赖**：M3 全链路；M1 补充语料（如需扩 TM）。
- **验收**：连续批跑 15 章无人工干预完成；单章成本 ≤ 预算（默认 ¥1.5/章）。

### M5 评测实验（W10–W11）

- **目标**：按 evaluation-plan.md 完成全套实验并产出论文用图表。
- **交付物**：
  - B1/B2/S 三系统测试集译文 + COMET/LLM-Judge 结果与显著性检验；
  - 5 变体消融矩阵结果；
  - MQM 人工标注（180 段）与 κ、加权错误分统计；
  - 实验数据包（快照 + 脚本）保证可复现。
- **依赖**：M4 冻结系统版本；标注人员与培训安排（W9 末提前预约）。
- **验收**：主表/消融表/案例齐备；至少一组对比达到统计显著。

### M6 论文与答辩（W12）

- **目标**：成文与展示。
- **交付物**：课程设计报告/毕业论文初稿（架构、实验、合规章节引用本文档族）；演示 demo（1 章全流程 + 回环可视化）；答辩 PPT。
- **依赖**：M5 全部实验结果。
- **验收**：报告通过组内互评；demo 现场可跑（含 `[DRAFT]` 兜底演示）。

## 3. MVP 压缩路径（M1 → M2 → M3 → M5）

当进度吃紧（如中期检查前必须出可演示成果）时，按以下压缩执行：

| 阶段 | 保留（MVP 范围） | 砍/降级 |
| --- | --- | --- |
| M1 | 1 部作品、长度对齐降级方案、术语 ≥ 100 条 | vecalign/LASER、多部作品、人工复核队列 |
| M2 | 完整保留（基线是对照组，必须有） | — |
| M3 | 六 Agent 全保留；记忆层只用 SQLite 术语库；回环上限 `max_rework=1` | FAISS/RAG（`search_tm` 返回空）、小说圣经简化为角色名列表 |
| M4 | **整体跳过**，只做 3 章冒烟测试 | 全部调优与成本看板 |
| M5 | 小规模：1 部作品 3 章；B2 与 S 对比 + 去术语库单变体消融；COMET + 简化 MQM（60 段） | B1、完整消融矩阵、LLM-as-Judge、三系统全量 |
| M6 | 照常（以 MVP 数据成文，注明范围局限） | — |

MVP 判定线：M3 末能演示"带一次 QA 回环的 1 章翻译"，M5 末能给出"S 优于 B2"的初步证据。

## 4. 风险与对策

| 风险 | 概率 | 影响 | 对策 |
| --- | --- | --- | --- |
| API key / 经费不足 | 中 | 高 | 全链路 `[DRAFT]` 占位可演示；fast/strong 混档与 TM 复用降本；申请课程经费或用本地开源模型兜底 |
| 语料版权不合规 | 中 | 高 | 只用公版/授权语料（data-pipeline.md 第 5 节）；M1 第一周完成合规审查 |
| 句对齐质量差 | 中 | 中 | vecalign 优先，长度对齐降级；低分入人工队列；验收线抽样准确率 95% |
| LangGraph/FAISS 环境装不上 | 中 | 中 | 延迟导入 + 降级路径是硬约定（numpy 检索替代 FAISS；状态机可退化为线性调用，回环由调度层手写） |
| QA 与翻译"互相不服"死循环 | 低 | 中 | `max_rework` 强制兜底 + `needs_human_review` 标记；M4 调通过线阈值 |
| 长上下文一致性仍差 | 中 | 中 | 圣经摘要 + 相邻段 `prev_segments` 注入；M4 对比实验量化增益，不足则在论文如实报告 |
| 并行开发接口漂移 | 中 | 高 | W5 冻结接口；architecture/agent-design 为唯一事实来源；改契约须全组同步并更新文档 |
| MQM 标注来不及 | 中 | 中 | W9 末预约标注员；压缩抽样量（180→60 段）保 κ 达标；最坏情况退化为 LLM-Judge + 小样本人工 |
| 评测不显著 | 低 | 中 | 预实验（M4 冒烟）先估效应量；扩大测试章数或聚焦术语一致性等强项维度呈现 |
| 关键成员掉线 | 低 | 高 | 每模块 AB 角；骨架先行保证他人可接手；文档与代码同仓 |

## 5. 里程碑检查点

- **W2 末**：M1 验收——JSONL 产物 + 质检记录。
- **W4 末**：M2 验收——基线译文 + 成本日志（中期检查 MVP 演示点①）。
- **W7 末**：M3 验收——端到端 demo + 回环触发录屏（中期检查 MVP 演示点②）。
- **W9 末**：M4 冻结——系统版本打 tag，进入评测期不再改功能。
- **W11 末**：M5 数据齐——全部图表可进论文。
- **W12**：M6 交付。
