# 评测方案（M5）

> 本文档定义 M5 阶段的实验评测设计：基线对比、消融实验矩阵、自动指标（COMET / LLM-as-Judge）、人工 MQM 抽样流程与统计口径。
> 目标：用可复现的实验回答两个问题——① 本系统是否显著优于基线？② 每个组件（审校 / 术语库 / 小说圣经）分别贡献多少？

---

## 1. 评测对象与测试集

### 1.1 受评系统

| 编号 | 系统 | 说明 |
| --- | --- | --- |
| B1 | 商用机翻 | Google Translate / DeepL（同一源文本，记录调用日期与版本） |
| B2 | 单 LLM 直译（M2 基线） | strong 档模型单 Prompt 直译，无术语库/圣经/审校/回环 |
| S | 本系统（全量） | 完整六 Agent + 记忆层 + QA 回环 |

### 1.2 测试集

- **规模**：从 M1 管道产出中选取 **3 部作品 × 各 5 章，每章约 2000 汉字**（共约 3 万字），覆盖修仙/玄幻/都市三种类型。
- **参考译文**：优先使用已出版官方译本（仅内部评测，合规见 data-pipeline.md 第 5 节）；无官方译本的章节由 2 名译者志愿翻译并互校。
- **预处理**：测试集章节从术语库/TM 的"记忆"中隔离（`work_id` 维度屏蔽），避免数据泄露造成虚高；训练侧数据仅来自其余章节。
- **固定快照**：测试集、术语库快照、配置（`workflow.max_rework` 等）打包存档，保证可复现。

## 2. 基线对比设计

- 三系统跑同一测试集，输出统一落盘为 `eval/runs/{system}/{work_id}/{chapter_id}.txt`。
- B1、B2 各运行 1 次（确定性输出或温度 0）；S 运行 3 次取指标均值以覆盖 QA 回环的随机性，同时记录 `rework_count` 分布与 token 成本。
- 对比维度：质量（第 4–6 节指标）+ 成本（¥/千字）+ 时延（分钟/章），论文中以"质量-成本"二维图呈现。

## 3. 消融实验矩阵

在 S（全量）基础上逐项移除组件，共 5 个变体：

| 变体 | 翻译 | 审校 | 润色 | QA 回环 | 术语库 | 小说圣经 | 验证的假设 |
| --- | :-: | :-: | :-: | :-: | :-: | :-: | --- |
| S-full | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | 基准 |
| S−审校 | ✓ | ✗ | ✓ | ✓ | ✓ | ✓ | 审校环节贡献准确性 |
| S−术语库 | ✓ | ✓ | ✓ | ✓ | ✗ | ✓ | 术语库贡献术语一致性 |
| S−圣经 | ✓ | ✓ | ✓ | ✓ | ✓ | ✗ | 圣经贡献长程连贯 |
| S−回环 | ✓ | ✓ | ✓ | ✗ | ✓ | ✓ | QA 回环贡献最终质量 |

- "去术语库"：清空 `glossary` 注入并禁用 `lookup_terms`（库仍保留但不生效）。
- "去圣经"：`get_story_bible` 返回空 `StoryBible`。
- "去审校"：状态机跳过 reviewer 节点；"去回环"：`max_rework=0`。
- 每个变体跑相同测试集，与 S-full 做配对比较；所有变体共用同一 LLM 档位配置，仅开关组件。

## 4. 自动指标一：COMET

- **工具**：`comet`（模型固定 `Unbabel/wmt22-comet-da`，记录版本号；延迟导入，未安装时跳过并提示）。
- **口径**：以段为单位打分，报告每系统**均值 ± 95% 置信区间**（按章 bootstrap 1000 次）。
- **解读**：COMET 需要参考译文，反映综合质量；对"术语一致性"不敏感，因此必须搭配 4.x 与 MQM。
- **显著性**：系统间做配对 bootstrap（段级配对），p < 0.05 判显著。

## 5. 自动指标二：LLM-as-Judge

- **动机**：部分测试章无参考译文，且 COMET 难评"风格/术语一致性"，引入强模型评审作为补充视角（同时注明其与 QA Agent 的独立性：评审用第三方模型/独立 Prompt，避免自我打分）。
- **维度与量表**（1–5 分，固定 rubric）：

  | 维度 | 锚点描述 |
  | --- | --- |
  | 准确性 | 1=大量误译漏译 … 5=忠实完整 |
  | 流畅性 | 1=难以卒读 … 5=母语者水准 |
  | 术语一致性 | 1=专名混乱 … 5=全书一致 |
  | 风格贴合 | 1=文体错位 … 5=类型感与语域到位 |

- **流程**：匿名化系统标签（sys_A/B/C 随机映射）→ 随机打乱呈现顺序 → 温度 0、逐维打分并要求一句理由 → 每份译文 2 次独立评审取均值；两次相差 ≥2 分的维度触发第三次评审仲裁。
- **口径**：按维度报告均值；汇总分 = 四维度等权平均。防偏见措施（匿名、乱序、独立评审）需在论文方法节声明。

## 6. 人工评测：MQM 抽样流程

### 6.1 抽样

- 分层随机抽样：每系统 × 每作品 × 每章随机抽 **4 段**，合计 3 系统 × 3 作品 × 5 章 × 4 段 = **180 段**。
- 段长 50–150 汉字；同一段的源文、参考译文（若有）、系统译文并排呈现，标注员不知系统身份。

### 6.2 标注规范（MQM 简化版）

| 类别 | 子类示例 | 严重度与权重 |
| --- | --- | --- |
| 准确性 Accuracy | 误译、漏译、增译 | minor ×1 / major ×5 / critical ×25 |
| 流畅性 Fluency | 语法、搭配、拼写、标点 | minor ×1 / major ×5 |
| 术语 Terminology | 与术语表/上下文不一致、错译 | minor ×1 / major ×5 |
| 风格 Style | 语域错位、文体不符 | minor ×1 / major ×5 |
| 格式 Format | 段落结构、标点规范 | minor ×1 |

- **双人独立标注**，分歧由第三人仲裁；标注前用 20 段样本培训并对齐标准。
- **一致性**：报告 Cohen's κ（按"错误类别 × 严重度"标签），κ ≥ 0.6 方可采信，不足则复盘标注指南重标。

### 6.3 统计口径

- 主指标：**加权错误分 / 千字** = Σ(错误数 × 权重) ÷ 总字数 × 1000，按系统 × 类别交叉报告。
- 系统间比较：段级配对，Wilcoxon 符号秩检验（样本量小、分布未知），p < 0.05 显著。
- 另报告 critical 错误密度（个/千字）作为"出版可用性"参考线。

## 7. 结果呈现与论文挂钩

1. 主表：三系统 × (COMET / LLM-Judge 四维 / MQM 加权分 / 成本 / 时延)。
2. 消融表：5 变体相对 S-full 的指标差值与显著性标记。
3. 案例研究：选 2–3 个典型段（术语一致性、回环修正前后对比）做定性分析。
4. 成本-质量权衡图：论证多 Agent 开销的收益区间（呼应 agent-design.md 第 5 节估算）。
5. 局限性声明：测试集规模、LLM-as-Judge 与 QA 同族模型风险、标注者主观性。
